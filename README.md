# VK Psychometrics Analyzer

Пайплайн для сбора и психометрического анализа профилей ВКонтакте с использованием ИИ.

## 📋 Описание

Этот проект состоит из 3-х отдельных скриптов, которые запускаются последовательно:

1. **step1_collect_profiles.py** - Сбор и фильтрация профилей из сообщества VK
2. **step2_analyze.py** - Психометрический анализ с помощью LLM (Qwen 2.5 72B)
3. **step3_generate_report.py** - Генерация красивого Markdown-отчета

Каждый скрипт экспортирует данные в JSON, что позволяет отлаживать пайплайн по шагам.

## 🔧 Установка

### 1. Установите зависимости

```bash
pip install vk_api requests
```

### 2. Создайте файл конфигурации

Скопируйте пример конфигурации:

```bash
cp config.json.example config.json
```

Заполните `config.json` своими данными:

```json
{
  "vk_user_token": "ваш_vk_токен",
  "huggingface_token": "ваш_hf_токен",
  "group_id": "id_сообщества",
  "max_profiles": 100
}
```

## 🔑 Получение токенов

### VK User Token

**Важно:** Для получения списка подписок пользователей нужен именно **User Token**, а не сервисный токен приложения.

#### Способ 1: Через Standalone-приложение (рекомендуется)

1. Зайдите в [VK Developers](https://dev.vk.com/)
2. Создайте новое приложение типа **Standalone**
3. Перейдите в настройки приложения
4. Вам понадобится **Service Token** или авторизуйте пользователя через OAuth

#### Способ 2: Быстрый способ через браузер

1. Откройте [https://oauth.vk.com/authorize?client_id=5168766&scope=groups,friends,photos,video,notes,pages,status,wall,ads,offline&response_type=token&display=page](https://oauth.vk.com/authorize?client_id=5168766&scope=groups,friends,photos,video,notes,pages,status,wall,ads,offline&response_type=token&display=page)
   - `client_id=5168766` - ID официального приложения VK
   - `scope` - запрашиваемые права (groups, friends, offline и др.)
   
2. Разрешите доступ приложению
3. После перенаправления вы увидите URL вида:
   ```
   https://oauth.vk.com/blank.html#access_token=ВАШ_ТОКЕН&expires_in=0&user_id=123456
   ```
4. Скопируйте значение `access_token` и вставьте в `config.json`

**Документация VK API:** https://dev.vk.com/ru/api/getting-started

#### Необходимые права доступа (scope):

- `groups` - доступ к сообществам
- `friends` - доступ к друзьям и подпискам
- `offline` - бессрочный токен

---

### Hugging Face Token

1. Зарегистрируйтесь на [Hugging Face](https://huggingface.co/)
2. Перейдите в [Settings → Tokens](https://huggingface.co/settings/tokens)
3. Нажмите **Create new token**
4. Дайте токену имя (например, "VK Analyzer")
5. Выберите роль **Read** (для доступа к API достаточно)
6. Скопируйте токен и вставьте в `config.json`

**Бесплатные лимиты Hugging Face Inference API:**
- ~30 запросов в минуту
- ~1000 запросов в день (зависит от модели)

## 🚀 Запуск пайплайна

### Шаг 1: Сбор профилей

```bash
python step1_collect_profiles.py
```

**Что делает:**
- Получает список участников указанного сообщества
- Фильтрует по полу (женский) и семейному положению (свободна)
- Проверяет открытость профиля
- Собирает расширенную информацию и подписки
- Сохраняет в `filtered_profiles.json`

**Выходные данные:** `filtered_profiles.json`

---

### Шаг 2: Психометрический анализ

```bash
python step2_analyze.py
```

**Что делает:**
- Загружает профили из `filtered_profiles.json`
- Отправляет каждый профиль в Qwen 2.5 72B через Hugging Face API
- Получает оценку по модели "Большой пятерки" и типу юмора
- Сохраняет результаты анализа

**Выходные данные:** `analyzed_profiles.json`

**Время выполнения:** ~2-3 секунды на профиль (зависит от лимитов API)

---

### Шаг 3: Генерация отчета

```bash
python step3_generate_report.py
```

**Что делает:**
- Загружает проанализированные профили
- Сортирует по совместимости
- Генерирует красивый Markdown-отчет с таблицами и деталями

**Выходные данные:** `result_report.md`

## 🎯 Критерии фильтрации

### Пол и статус отношений
- Только женский пол (`sex == 1`)
- Свободна (исключаются `relation`: 2-есть друг, 3-помолвлена, 4-замужем, 7-влюблена, 8-гражданский брак)

### Открытость профиля
- Профиль должен быть открыт (`is_closed == false`)
- Должен быть доступен `can_access_closed == true`

### Психометрические критерии (LLM анализ)
- **Экстраверсия:** активность, социальные маркеры
- **Нейротизм:** эмоциональная стабильность (низкий = хорошо)
- **Тип юмора:** аффилиативный или самоирония (плюс), агрессивный (минус)
- **Открытость опыту:** путешествия, творчество, мастер-классы
- **Добросовестность:** стабильность, надежность

## 🛠 Отладка

### Если шаг 1 не работает:
- Проверьте правильность `vk_user_token`
- Убедитесь, что вы состоите в указанном сообществе
- Проверьте, что `group_id` указан верно (можно использовать короткое имя или числовой ID)

### Если шаг 2 не работает:
- Проверьте `huggingface_token`
- Убедитесь, что `filtered_profiles.json` существует
- При ошибке 503 (модель загружается) - скрипт автоматически подождет 30 секунд

### Если шаг 3 не работает:
- Убедитесь, что `analyzed_profiles.json` существует
- Проверьте формат JSON файла

## 📄 Формат отчета

Отчет `result_report.md` содержит:

1. **Шапка** - дата, источник, модель анализа
2. **Статистика** - количество обработанных профилей
3. **Топ кандидатов** - таблица с лучшими совпадениями (score ≥ 80)
4. **Детальный разбор** - подробный анализ каждого топ-кандидата
5. **Техническая статистика** - распределение по скорам

Отчет можно открыть в любом Markdown-просмотрщике или загрузить на GitHub.

## ⚠️ Ограничения

- API VK бесплатное, но имеет лимиты (~3 запроса в секунду)
- Hugging Face API бесплатный с ограничениями (~30 запросов/минуту)
- Анализ возможен только для открытых профилей
- Психометрическая оценка приблизительная и требует ручной проверки
